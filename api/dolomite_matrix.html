dolomite_matrix.choose_dense_chunk_sizes module#

dolomite_matrix.choose_dense_chunk_sizes.choose_dense_chunk_sizes(shape, size, min_extent=100, memory=10000000.0)[source]#

Choose some chunk sizes to use for a dense HDF5 dataset. For each dimension, we consider a slice of the array that consists of the full extent of all other dimensions. We want this slice to occupy less than memory in memory, and we resize the slice along the current dimension to achieve this. The chosen chunk size is then defined as the size of the slice along the current dimension. This ensures that efficient iteration along each dimension will not use any more than memory bytes.

Parameters:

  • shape (Tuple[int, ...]) – Shape of the array.

  • size (int) – Size of each array element in bytes.

  • min_extent (int) – Minimum extent of each chunk dimension, to avoid problems with excessively small chunk sizes when the data is large.

  • memory (int) – Size of the (conceptual) memory buffer to use for storing blocks of data during iteration through the array, in bytes.

Return type:

Tuple[int, ...]

Returns:

Tuple containing the chunk dimensions.

dolomite_matrix.stage_DelayedArray module#

dolomite_matrix.stage_DelayedArray.stage_DelayedArray(x, dir, path, is_child=False, chunks=None, cache_size=100000000.0, block_size=100000000.0, **kwargs)[source]#

Method for saving DelayedArray objects to file, see stage_object() for details.

Parameters:

  • x (DelayedArray) – Array to be saved.

  • dir (str) – Staging directory.

  • path (str) – Relative path inside dir to save the object.

  • is_child (bool) – Is x a child of another object?

  • chunks (Union[Tuple[int, ...], int, None]) –

    For dense x, a tuple of chunk dimensions. If None, we automatically choose some chunk sizes with choose_dense_chunk_sizes().

    For sparse x, an integer specifying the chunk dimension in write_sparse_matrix(). If None, a suitable chunk size is chosen.

  • cache_size (int) – Size of the HDF5 cache size, in bytes. Larger values improve speed at the cost of memory.

  • block_size (int) – Size of each block in bytes. Staging is performed by iterating over x, extracting one block at a time, and saving it to the HDF5 file. Larger values improve speed at the cost of memory.

  • kwargs – Further arguments, ignored for dense x. For sparse x, these are passed to write_sparse_matrix().

Return type:

dict[str, Any]

Returns:

Metadata that can be edited by calling methods and then saved with write_metadata().

dolomite_matrix.write_sparse_matrix module#

dolomite_matrix.write_sparse_matrix.write_sparse_matrix(x, path, name, chunks=None, guess_integer=True, block_size=100000000.0)[source]#

Write a sparse matrix into a HDF5 file in a compressed-sparse column format. This creates a HDF5 group containing the data, indices and indptr datasets, containing the corresponding components of the usual compressed sparse representation. The shape of the matrix is stored in the shape dataset inside the same group.

Parameters:

  • x – Any sparse matrix.

  • path (str) – Path to the file in which to save x.

  • name (str) – Name of the group in the file.

  • chunks (Optional[int]) – Size of the 1-dimensional chunks for the data and index datasets. If None, defaults to 10000.

  • guess_integer (bool) – Whether to guess a compact integer type for the data. This can reduce disk usage at the cost of a second pass through the data.

  • block_size (int) – Size of the blocks to use for processing x. Larger values increase speed at the cost of memory efficiency.

Returns:

A HDF5 file is created at path with the sparse data stored in the name group.
